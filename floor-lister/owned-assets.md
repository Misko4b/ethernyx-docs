# Owned Assets

The **Owned Assets** tab pulls in **every NFT across all your wallets**, so you can list anything you're holding without hunting for contract addresses.

<div align="left"><figure><img src="../.gitbook/assets/image (6).png" alt=""><figcaption></figcaption></figure></div>

Each asset card shows:

* **Image and name** (falls back to `collection #token_id` if the NFT has no name).
* **Collection** — click it to filter the list.
* **Rarity / rank** — colored by tier (Legendary, Epic, Rare).
* **Last sale** — the most recent sale price, or "No sales."
* **Chain**, **token standard** (ERC-721 / ERC-1155), **token ID**, and the **wallet** that holds it.

## Filtering

* **Search** by name, collection slug, or token ID.
* Filter by **All**, **With Task**, or **No Task** — so you can instantly see what you're not listing yet.
* Toggle **grid** or **list** view.

## One-click task creation

* **Create Task** — on any asset without a task. It opens the listing form with the collection, token ID, and wallet already filled in — just set your minimum price and go. → [Creating a Listing](creating-a-listing.md)
* **Go to Task** — if an asset already has a listing task, this jumps straight to it in the [Tasks](managing-tasks.md) tab.
